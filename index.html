Web front-end performance monitoring

This project aims to automate the monitoring or your website load time from an end-user point of view (with multiple synthetic tests). It uses PhantomJS to access webpages, and record the time it takes to download each individual requests using the HAR format. This analysis report is then saved to a database to later aggregate more informations about the load time for the page and the individual requests.

Requisites

  • MongoDB >= 2.2
  • PHP >= 5.3.2 (and extensions: mongo)
  • Python (and libs: pymongo, celery)
  • PhantomJS >= 1.5
  • RabbitMQ
  • A Web server (apache, nginx)

Why create another tool for this?

Many tools already exists online, as services, and even open source projects. But many of them are not giving enough informations about individual assets, are providing only the result of one individual page load, or are just giving the average load time for multiple requests. The main objective of this tool is to provide monitoring of your pages over time, by loading the same page multiple times, discarding the extreme values, and returning a stable significant value of the front-end performance of your page, in an automated way, ie continuous load time performance testing.

Also the customisation of existing tools is quite difficult, thus starting a project from scratch with a simple interface and code makes it very flexible to add functionalities.

Google Analytics and NewRelic are good at giving you Real User Measurement, that tells you how your website performs in the real world. This is not what this tool is intending to monitor. It is mainly to track the performance from one point of view and make sure that any optimisation or regular development (features) are not degrading the load time of your site. Regardless of if you are using caching or not, it will provide you a good idea of how slow your pages are.
